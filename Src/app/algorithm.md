# 🏆 グループ3 アルゴリズム実装要約

### 💡 核心となる基本方針
**「送信機（TX）はペナルティ0点」「受信機（RX）だけが-5点対象」** というルール特性を100%活かした **2ノード協調型サバイバルアルゴリズム** です。

```
 [送信機 TX Node (0x0003)]                         [受信機 RX Node (0x0007)]
  - ペナルティ対象外 (Q1)                             - 被弾ペナルティ-5点対象
─────────────────────────────────────────────────────────────────────────────
 1. SF7で公式50Bパケット送信 (+1点)                  1. CH1 / SF7で常時RX待機
 2. 600ms休止(ToA×9)中、安全にRXスカウト             │
 3. 混雑/他チーム検出時:                            │
    - 制御パケット "GRP3:CMD_CH7" 送信 ──────────► 2. 制御パケットを受信！
    - 自身もCH7へホップ                                - 即座にCH7へホップ（同期100%）
                                                    │
                                                    3. 万が一ライバル受信時(-5点被弾):
                                                       - 即座にSF10ステルスへ500ms避難
```

---

## 🔑 アルゴリズムの 5 大ポイント

### 1. 2ノード完全自動同期 ＆ 安全誘導（Q3の活用）
* 送信機がLBTブロック（3回連続失敗）やスカウトでチャンネル変更（CH1 ↔ CH7）を決めた際、受信用に**制御パケット（`GRP3:CMD_CH7` など）をブロードキャスト送信**します。
* 制御パケットは公式50Bペイロードではないため、**他チームにペナルティを与えずに自チームの受信機だけを完璧に同期・誘導**できます。

### 2. 送信機による「ペナルティ0点」安全スカウト（Q1の活用）
* 送信機はパケット送信後の600ms休止時間（Duty Rest）中、**安全にRXモードで空間をスキャン**します。
* 送信機がライバルパケットを受信してもペナルティは0点のため、リスクゼロで周囲の電波混雑状況を学習・回避できます。

### 3. SF10ステルス避難 ＆ 電波法（DC制約）の100%遵守
* 受信機がライバルのパケットを受信（被弾）した直後、自動的に**SF10（ステルスモード）**へ500ms退避します。SFの数学的直交性により、ライバルのSF7パケットを透過して後続の連続被弾を防ぎます。
* 送信休止時間（Duty Rest）は `ToA × 9`（約540ms）に対して **600ms** をタイマー設定。さらにファームウェア内部の `PhyDutyTimer` ガードもあるため、**DC制約・電波法遵守も100%保証**されます。

### 5. 10分間（600秒）自動停止＆スコア集計システム【NEW】
* 競技開始から10分（600,000ms）経過した瞬間、**プログラムが自動で動作を完全停止**します。
* 終了時に、**推定グループスコアの集計表をターミナルに自動出力**します。
* 停止後は**RXモードも完全にクローズ（Standby状態に固定）**されるため、競技終了後の時間外被弾ペナルティ（他チームがまだ送信している場合の減点）を完璧に防ぎます。

## 🏅 技術課題ボーナスクリアの根拠（計 +100点）

1. **自律チャネルホッピング (+50点)**:
   LBTブロック3回またはスカウト検知でCH1 ↔ CH7へ自律移動。  
   ログ出力例: `[TX_NODE] CH_HOP trigger=LBT_THRESHOLD from=CH1 to=CH7`
2. **SF直交性回避 / ステルスモード (+50点)**:
   被弾時にSF10へ一時避難（500ms）。  
   ログ出力例: `[RX_NODE] Entering SF10 Stealth for 500 ms`
